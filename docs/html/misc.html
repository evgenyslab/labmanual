10.1. Mount Remote Directory Locally in Linux¶

In the case of remote development, it may be beneficial to mount the remote’s development path on the local computer, use a local file editor to make and save changes, and then compile and run remotely.

In linux, to mount a remote directory from a remote machine, use sshfs:

sudo apt-get install sshfs

Then, it is neccesary to create a local directory into which the remote will mount:

mkdir /my/local/mount/point

Finally, use sshfs to mount the remote directory into the newly created local:

sudo sshfs -o allow_other remote_user@10.10.10.10:/remote/path/to/mount/ /my/local/mount/point

10.2. Linux / Mac Splitting TAR Files¶

To create tar archive:

# first tar the folder:
tar -cvzf tarFileName.tar.gz /Directory/To/Tar/*

# then split the folder:
split -b 2000m tarFileName.tar.gz "tarFileName.tar.gz.part"

# to recombine:
cat tarFileName.tar.gz.part* > tarFileName.tar.gz

10.3. Local app install¶

Application executibles can be installed by placing a sh script to launch the application under $HOME/.local/bin, this will be visible to the current user.

10.4. Sphinx & GitHub Pages¶

Source: [ref].

IMPORTANT NOTE: github uses jekyll to generate pages, however, standard sphinx doc structure has a _static folder generate with all css and js files. When Github tried to compile the site with jekyll, files/folders with _ are ignored.

THEREFORE when using the gh-pages branch side-by-side repo config as per the instructions in the ref, it is important to add .nojekyll in the root directory. This allows style sheets and js to work correctly.

10.4.1. Drawio w/ Sphinx¶

see: https://pypi.org/project/sphinxcontrib-drawio/

To get this to work, need to make sure sphinxcontrib-drawio to the extensions in conf.py for docs.

Then, need to add the application executible for sphinx to render the drawing by adding drawio_binary_path='/path/to/app' in the conf.py configuraiton file.

Then, Images can be rendered and exported using a modified image directive.

Note, drawio page id can be used to specify which page of the diagram to export.

The drawio diagrams can be stored in a relative location to source, during document compilation, drawio will be used to render the image and place it in appropriate folder under html build directory.
